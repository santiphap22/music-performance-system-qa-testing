# Use Case Specification: Login

| Use Case Attribute | Specification Details |
| :--- | :--- |
| **1. Use Case ID** | 1 |
| **2. Use Case Name** | Login |
| **3. Primary Actor** | User ทุกประเภท |
| **4. Description** | User สามารถเข้าสู่ระบบเพื่อใช้งานแอปพลิเคชันตามสิทธิ์ของตนเองได้ |
| **5. Trigger** | User ต้องการเข้าสู่ระบบเพื่อใช้งานแอปพลิเคชัน |
| **6. Preconditions** | PRE-1 User ต้องมีบัญชี Discord เป็นของตัวเอง<br> PRE-2System admin ต้องนำข้อมูล Google Forms ที่สมาชิกกรอกตอนสมัครเข้า SCI BAND เข้าสู่ระบบเพื่อให้ผู้ใช้สามารถเข้าสู่ระบบได้ |
| **7. Postconditions** | POST-1 User เข้าสู่ระบบและใช้งานแอปพลิเคชันได้ตามสิทธิ์ของตนเอง |
| **11. Priority** | High |
| **12. Frequency of Use** | 1 ครั้งต่อ 1 คน |

---

### Workflows & Flow of Events

#### 8. Normal Flow
1. User กดเข้าสู่ระบบ
2. User กรอก Gmail หรือเบอร์มือถือและรหัสผ่าน
3. User กดยืนยันเพื่อเข้าสู่ระบบ
4. User เข้าสู่ระบบสำเร็จและระบบนำผู้ใช้ไปยังหน้าเมนูหลักพร้อมกับสามารถใช้งานระบบได้ตามสิทธิ์ของผู้ใช้

#### 9. Alternative Flow
* User มีแอปพลิเคชัน Discord และทำการเข้าสู่ระบบไว้แล้ว 
    1. User กดเข้าสู่ระบบ 
    2. User กดยืนยันเพื่อเข้าสู่ระบบ 
    3. Userเข้าสู่ระบบสำเร็จและระบบนำผู้ใช้ไปยังหน้าเมนูหลักพร้อมกับสามารถใช้งานระบบได้ตามสิทธิ์ของผู้ใช้ 

#### 10. Exceptions
1. User กดเข้าสู่ระบบ
2. User กรอก Gmail หรือเบอร์มือถือและรหัสผ่าน
3. User กดยืนยันเพื่อเข้าสู่ระบบ
    - ระบบขึ้นข้อความแจ้งเตือนว่า "คุณไม่มีสิทธิ์ในการเข้าใช้งานระบบนี้"
4. User แจ้ง System admin หรือ Staff ที่เกี่ยวข้อง
5. System admin ทำการเพิ่มบัญชีผู้ใช้เข้าสู่ระบบ
6. User เข้าสู่ระบบอีกครั้ง
7. Use case ดำเนินตามปกติ
