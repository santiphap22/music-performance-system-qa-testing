# Use Case Specification: รายละเอียดกรณีดูรายละเอียดงานแสดงดนตรี

| Use Case Attribute | Specification Details |
| :--- | :--- |
| **1. Use Case ID** | 2 |
| **2. Use Case Name** | ดูรายละเอียดงานแสดงดนตรี |
| **3. Primary Actor** | User ทุกประเภท |
| **4. Description** | User ทุกคนสามารถดูรายละเอียดงานแสดงดนตรีแต่ละงาน เพื่อทราบถึงข้อมูลงานแสดงนั้นๆได้ |
| **5. Trigger** | User ต้องการดูข้อมูลงานแสดงดนตรีเพื่อทราบถึงรายละเอียดของงานนั้นๆ |
| **6. Preconditions** | PRE-1 Backstage มีการเพิ่มงานแสดงดนตรีลงไปในระบบแล้ว<br> PRE-2 User ทำการยืนยันตัวตนสำเร็จ |
| **7. Postconditions** | POST-1 User ดูรายละเอียดงานแสดงดนตรีสำเร็จ |
| **11. Priority** | Medium |
| **12. Frequency of Use** | 1-10 ครั้งต่อหนึ่งงานดนตรี |

---

### Workflows & Flow of Events

#### 8. Normal Flow
1. User เลือก "Event" จากหน้า Main Menu 
    - ระบบแสดงรายการงานแสดงดนตรีทั้งหมดที่เกี่ยวข้อง 
2. User เลือกงานแสดงดนตรีจากรายการงานแสดงดนตรีที่มี 
    - ระบบทำการแสดงหน้ารายละเอียดของงานแสดงดนตรีที่เลือก โดยประกอบไปด้วย ชื่อ วันเวลาแสดง Dress code และรายละเอียดเพิ่มเติม 
3. หากต้องการดูงานแสดงดนตรีอื่นๆ 
    - User กดย้อนกลับไปยังหน้า Event 
    - User เลือกดูรายละเอียดงานแสดงดนตรีที่ต้องการ 
4. Use case สิ้นสุดเมื่อ user ไม่ต้องการดูงานแสดงดนตรีอื่น ๆ แล้ว

#### 9. Alternative Flow
- None

#### 10. Exceptions
**Event ยังไม่ได้ถูกเพิ่มลงไปในระบบ**
1. User เลือก "Event" จากหน้า Main Menu 
    - ระบบแสดงรายการงานแสดงดนตรีทั้งหมดที่เกี่ยวข้อง 
2. User ไม่พบงานแสดงดนตรีที่ต้องการในระบบ User ต้องแจ้งปัญหาให้ Backstage 
3. User ทำการแจ้งปัญหาไปที่ Backstage ผ่านช่องทางส่วนตัว 
4. Backstage ดำเนินการตาม **Use case สร้างงานแสดงดนตรี** เพื่อเพิ่มEventที่ต้องการ 
    - ระบบบันทึกงานแสดงดนตรีใหม่และแสดงในรายการ 
5. User เข้ามาดูรายละเอียดงานแสดงดนตรีอีกครั้ง 
6. Use Case ดำเนินการตามปกติ
