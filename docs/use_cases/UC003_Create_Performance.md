| :--- | :--- |
| **1. Use Case ID** | 3 |
| **2. Use Case Name** | สร้างงานแสดงดนตรี |
| **3. Primary Actor** | Backstage |
| **4. Description** | Backstage สามารถสร้างงานแสดงดนตรีขึ้นมาเพื่อนำข้อมูลเข้าสู่ระบบและเพื่อให้ผู้ใช้ทุกคนสามารถเข้ามาดูรายละเอียดได้ |

### Workflows & Flow of Events

#### 8. Normal Flow
1. Backstage เลือก "Event" จากหน้า Main Menu 
    - ระบบแสดงรายการงานแสดงดนตรีทั้งหมด 
2. Backstage กดไอคอนรูปเครื่องหมาย (+) จากหน้า Event 
    - ระบบแสดงฟอร์มสำหรับเพิ่มงานแสดงดนตรีใหม่โดยใส่ชื่อ งานแสดงดนตรี, วันเดือนปีที่จัดแสดง , เวลาเริ่มงาน-เวลาสิ้นสุดงาน,การแต่งกายวันงาน (dress code) และรายละเอียดเพิ่มเติม 
3. Backstage กรอกข้อมูลที่ต้องการและกดยืนยัน 
    - ระบบบันทึกข้อมูล Event ใหม่ลงในฐานข้อมูล 
    - ระบบแสดงข้อความแจ้งเตือน: "สร้าง Event สำเร็จ" 
4. ระบบกลับไปที่หน้ารายการ Event 
5. หากต้องการเพิ่มงานแสดงดนตรีอื่นๆ 
    - ระบบนำ Backstage กลับไปยังหน้างานแสดงดนตรีทั้งหมด 
6. Backstage ทำซ้ำขั้นตอน 2 - 5 
7. Use case สิ้นสุดเมื่อ Backstage ไม่ต้องการเพิ่มงานแสดงดนตรีอื่น ๆ แล้ว

#### 9. Alternative Flow
**Backstage ต้องการยกเลิกการเพิ่มงานแสดงดนตรี**
1. ขณะที่อยู่ในฟอร์มเพิ่มงานแสดงดนตรีใหม่ 
2. Backstage กดปุ่ม x เพื่อยกเลิกการสร้างงานแสดงดนตรี 
    - ระบบขึ้นข้อความให้ backstage ว่า "คุณต้องการยกเลิกการสร้าง event หรือไม่" พร้อมปุ่มยืนยันและยกเลิก 
3. Backstage กดยืนยัน 
    - ระบบนำ Backstage กลับไปยังหน้ารายการงานแสดงดนตรีทั้งหมด 
4. Alternative Flow สิ้นสุด

#### 10. Exceptions
**ข้อมูลที่กรอกไม่ครบถ้วนหรือไม่ถูกต้อง**
1. ขณะที่ Backstage กรอกข้อมูลงานแสดงดนตรีใหม่ 
2. ระบบตรวจพบข้อผิดพลาด เช่น: 
    - ชื่อ Event หรือ Dress Code เป็นค่าว่าง 
3. ระบบแสดงข้อความแจ้งเตือน: 
    - “กรุณากรอกชื่อ Event” หรือ “กรุณากรอก Dress Code”  
4. Backstage กรอกข้อมูลให้ถูกต้อง 
5. Backstage กดปุ่ม "สร้าง" เพื่อปิดข้อความแจ้งเตือน: 
    - ระบบขึ้นข้อความให้ backstage ว่า "สร้าง Event สำเร็จ"
6. Use case ดำเนินตามปกติ
