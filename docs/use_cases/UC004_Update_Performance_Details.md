# Use Case Specification: รายละเอียดกรณีแก้ไขงานแสดงดนตรี

| Use Case Attribute | Specification Details |
| :--- | :--- |
| **1. Use Case ID** | 4 |
| **2. Use Case Name** | แก้ไขรายละเอียดงานแสดงดนตรี |
| **3. Primary Actor** | Backstage |
| **4. Description** | Backstage สามารถการแก้ไขรายละเอียดงานแสดงดนตรีที่สร้างเอาไว้ เช่น ชื่อ วันเวลา Dress Code และ รายละเอียดเพิ่มเติม |
| **5. Trigger** | Backstage ต้องการแก้ไขรายละเอียดข้อมูล Event เนื่องจากมีการเปลี่ยนแปลงของข้อมูล |
| **6. Preconditions** | PRE-1 Backstage ทำการยืนยันตัวตนสำเร็จ<br> PRE-2 ในงานแสดงดนตรีนั้น ๆ จะต้องมีการตกลงกันก่อนว่าจะปรับเปลี่ยนข้อมูลอะไรบ้าง |
| **7. Postconditions** | POST-1 งานแสดงดนตรีในระบบถูกแก้ไข |
| **11. Priority** | High |
| **12. Frequency of Use** | 0-5 ครั้งต่อหนึ่งงานดนตรี |

### Workflows & Flow of Events

#### 8. Normal Flow
1. Backstage เลือก "Event" จากหน้า Main Menu 
    - ระบบแสดงรายการงานแสดงดนตรีทั้งหมด 
2. Backstage เลือกงานแสดงดนตรีที่ต้องการจากรายการงานแสดงดนตรีที่มี 
    - ระบบทำการแสดงหน้ารายละเอียดของงานแสดงดนตรีที่เลือก
3. Backstage ทำการแก้ไขรายละเอียดโดยสามารถแก้ไขชื่องานแสดงดนตรี, วันเวลา, การแต่งกายวันงาน (dress code) และรายละเอียดเพิ่มเติม 
4. Backstage กดยืนยันการแก้ไข 
    - ระบบแสดงข้อความแจ้งเตือน: "อัปเดต Event สำเร็จ" 
5. หากต้องการแก้ไขงานแสดงดนตรีอื่นๆเพิ่มเติม 
    - ระบบนำ Backstage กลับไปยังหน้าเลือกงานแสดงดนตรี 
6. Backstage ทำซ้ำขั้นตอนที่ 2 - 5 
7. Use Case สิ้นสุดเมื่อ Backstage ไม่ต้องการแก้ไขงานแสดงดนตรีอื่น ๆ เพิ่มเติม

#### 9. Alternative Flow
**Backstage ต้องการยกเลิกการแก้ไข**
1. ขณะที่ Backstage อยู่ในฟอร์มแก้ไขรายละเอียดงานแสดงดนตรี 
2. Backstage กดย้อนกลับไปที่หน้า Event ทั้งหมด 
3. Alternative flow สิ้นสุด 

#### 10. Exceptions
**Backstage แก้ไขข้อมูลไม่ถูกต้อง**
1. ขณะที่ Backstage กรอกข้อมูลในฟอร์มแก้ไขรายละเอียดงานแสดงดนตรี 
2. ระบบตรวจพบข้อผิดพลาด เช่น: 
    - ชื่อ Event หรือ Dress Code เป็นค่าว่าง  
3. ระบบแสดงข้อความแจ้งเตือน:
    - “กรุณากรอกชื่อ Event” หรือ “กรุณากรอก Dress Code”  
4. Backstage กดปุ่ม "อัปเดต" เพื่อปิดข้อความแจ้งเตือน 
    - ระบบขึ้นข้อความให้ backstage ว่า “อัปเดต Event สำเร็จ” 
5. Exception Flow สิ้นสุด หลังจาก Backstage กรอกข้อมูลครบถ้วนและกดยืนยันอีกครั้ง
