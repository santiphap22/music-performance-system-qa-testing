# Use Case Specification: รายละเอียดกรณีดู queue เพลง

| Use Case Attribute | Specification Details |
| :--- | :--- |
| **1. Use Case ID** | 6 |
| **2. Use Case Name** | ดู queue เพลง |
| **3. Primary Actor** | User ทุกประเภท |
| **4. Description** | User สามารถดูคิวเพลงในแต่ละงานแสดงดนตรีได้ |
| **5. Trigger** | User ต้องการดู queue เพลงในงานแสดงดนตรีนั้น ๆ |
| **6. Preconditions** | PRE-1 Backstage มีการเพิ่มงานแสดงดนตรีลงไปในระบบแล้ว<br> PRE-2 Backstage ได้เพิ่มเพลงลงไปในงานแสดงดนตรีนั้น ๆแล้ว<br> PRE-3 User ทำการยืนยันตัวตนสำเร็จ |
| **7. Postconditions** | POST-1 User ดู queue เพลงสำเร็จ |
| **11. Priority** | High |
| **12. Frequency of Use** | 1-10 ครั้งต่อหนึ่งงานดนตร |

---

### Workflows & Flow of Events

#### 8. Normal Flow
1. User เลือก "Event" จากหน้า Main Menu 
    - ระบบแสดงรายการงานแสดงดนตรีทั้งหมดที่เกี่ยวข้อง 
2. User เลือกงานแสดงดนตรีจากรายการที่มี
    - ระบบทำการแสดงหน้ารายละเอียดของงานแสดงดนตรีที่เลือก 
3. User กดปุ่ม "ดูรายชื่อเพลง" 
    - ระบบแสดงรายชื่อเพลงที่ใช้ในการแสดงของงานแสดงดนตรีนั้น ๆ 
4. หากต้องการดู queue เพลงของงานแสดงดนตรีอื่น ๆ 
    - User กด tab bar “Event ทั้งหมด” จากหน้ารายละเอียดเพลง 
    - ระบบนำ User กลับไปยังหน้าเลือก Eventทั้งหมด 
5. User ทำซ้ำขั้นตอน 2 - 4 
6. Use case สิ้นสุดเมื่อ user ไม่ต้องการดู Event อื่น ๆ แล้ว

#### 9. Alternative Flow
**Performer ต้องการดู queue เพลงเฉพาะเพลงที่ตนเองมีส่วนร่วม**
1. Performer เลือก "Event" จากหน้า Main Menu 
    - ระบบแสดงรายการงานแสดงดนตรีทั้งหมดที่เกี่ยวข้อง 
2. Performer เลือกงานงานแสดงดนตรีจากรายการที่มี 
    - ระบบทำการแสดงหน้ารายละเอียดของงานแสดงดนตรีที่เลือก 
3. Performer กดปุ่ม "ดูรายชื่อเพลง" 
    - ระบบแสดงรายชื่อเพลงที่ใช้ในการแสดงของงานแสดงดนตรีนั้น 
4. Performer ทำการกดปุ่ม "แสดงเฉพาะเพลงของฉัน" 
    - ระบบทำการกรองเฉพาะเพลงที่ Performer มีส่วนเกี่ยวข้อง 
5. หากต้องการดู queue เพลงของ Event อื่นๆ 
    - User กด tab bar “Event ทั้งหมด” จากหน้ารายละเอียดเพลง 
    - ระบบนำ User กลับไปยังหน้าเลือก Eventทั้งหมด 
6. User ทำซ้ำขั้นตอน 2 - 5 
7. Use case สิ้นสุดเมื่อ userไม่ต้องการดูEventอื่นๆแล้ว 
    - User กดไอคอนรูป (Event) จากหน้ารายละเอียดเพลง 
    - ระบบนำ User กลับไปยังหน้าเลือกงานแสดงดนตรีทั้งหมด 
8. User ทำซ้ำขั้นตอน 2 – 4 
9. Use case สิ้นสุดเมื่อ User ไม่ต้องการดู queue เพลง อื่น ๆ แล้ว

#### 10. Exceptions
**เพลงยังไม่ได้ถูกเพิ่มลงไปในระบบ**
1. User เลือก "Event" จากหน้า Main Menu 
    - ระบบแสดงรายการงานแสดงดนตรีทั้งหมดที่เกี่ยวข้อง
2. User เลือกงานแสดงดนตรีจากรายการที่มี 
    - ระบบทำการแสดงหน้ารายละเอียดของ Event ที่เลือก 
3. User กดปุ่ม "ดูรายชื่อเพลง" 
4. User ไม่พบเพลงที่ต้องการ 
    - User ทำการแจ้งปัญหาไปที่ Backstage ผ่านช่องทางส่วนตัว 
5. Backstage ดำเนินการตาม **Use case เพิ่มเพลงใน queue** เพื่อเพิ่มเพลงที่ต้องการ 
    - ระบบบันทึก queue เพลงใหม่และแสดงในรายการ 
6. Use Case ดำเนินการตามปกติ
