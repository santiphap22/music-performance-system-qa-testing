# Use Case Specification: รายละเอียดกรณีดูรายละเอียดเพลง

| Use Case Attribute | Specification Details |
| :--- | :--- |
| **1. Use Case ID** | 11 |
| **2. Use Case Name** | ดูรายละเอียดเพลง |
| **3. Primary Actor** | User ทุกประเภท |
| **4. Description** | User ทุกประเภทสามารถดูรายละเอียดในแต่ละเพลงได้ว่าเพลงนั้นมี Performerคนไหนเล่นบ้าง รวมถึง ชื่อเพลง, คีย์เพลง และอ้างอิงเพลง (reference) |
| **5. Trigger** | User ต้องการดูข้อมูลของแต่ละเพลง |
| **6. Preconditions** | PRE-1 Backstage มีการเพิ่มงานแสดงดนตรีลงไปในระบบแล้ว <br> PRE-2 Backstage ได้เพิ่มเพลงลงไปในงานแสดงดนตรีนั้น ๆแล้ว<br> PRE-3 Backstage ได้ assign เพลงให้กับ Performer แล้ว<br> PRE-4 User ยืนยันตัวตนสำเร็จ |
| **7. Postconditions** | POST-1 User ดูรายละเอียดเพลงสำเร็จ |
| **11. Priority** | High |
| **12. Frequency of Use** | 1-10 ครั้งต่อหนึ่งงานดนตรี |

### Workflows & Flow of Events

#### 8. Normal Flow
1. User เลือก "Event" จากหน้า Main Menu 
    - ระบบแสดงรายการงานแสดงดนตรีทั้งหมด 
2. User เลือกงานงานแสดงดนตรีจากรายการที่มี 
    - ระบบทำการแสดงหน้ารายละเอียดของ งานแสดงดนตรีที่เลือก 
3. User กดปุ่ม "ดูรายชื่อเพลง" 
    - ระบบแสดงรายชื่อเพลงที่ใช้ในการแสดงของงานแสดงดนตรีนั้น 
4. User เลือกเพลงที่ต้องการดูรายละเอียดเพลง 
5. ระบบทำการแสดงหน้ารายละเอียดของเพลงโดยจะมี ชื่อเพลง จำนวน performer ในแต่ละตำแหน่ง , คีย์เพลง (Song Key) , อ้างอิงเพลง (reference) 
6. หาก User ต้องการดู lineup performer นั ้น User สามารถกดปุ่ม “ดูข้อมูล” ตามตำแหน่ง performer ที่ต้องการและระบบจะแสดงชื่อ Performer ที่เกี่ยวข้อง 
7. หากต้องการดูรายละเอียดเพลงอื่นๆ 
    - User กดย้อนกลับ
    - ระบบนำ User กลับไปยังหน้ารายชื่อเพลงของงานแสดงดนตรีนั้น ๆ   
8. User ทำซ้ำขั้นตอน 4 - 6 
9. Use case สิ้นสุดเมื่อ user ไม่ต้องการดูเพลงอื่น ๆแล้ว

#### 9. Alternative Flow
**Performer ต้องการดูรายละเอียดเพลงเฉพาะเพลงที่ตนเองเกี่ยวข้อง** 
1. Performer เลือก "Event" จากหน้า Main Menu 
    - ระบบแสดงรายการงานแสดงดนตรี ทั้งหมดที่เกี่ยวข้อง 
2. Performer เลือกงานงานแสดงดนตรีจากรายการงานแสดงดนตรีที่มี 
3. Performer กดปุ่ม "ดูรายชื่อเพลง" 
    - ระบบแสดงรายชื่อเพลงที่ใช้ในการแสดงของงานแสดงดนตรีนั้น 
4. Performer กด checkbox ตรง "แสดงเฉพาะเพลงของฉัน" 
    - ระบบทำการกรองเพลงให้ Performer 
5. Performer เลือกเพลงที่ต้องการดูรายละเอียดเพลง 
6. ระบบทำการแสดงหน้ารายละเอียดของเพลงโดยจะมี ชื่อเพลง จำนวน performer ในแต่ละตำแหน่ง , คีย์เพลง (Song Key) , อ้างอิงเพลง (reference) 
7. หาก Performer ต้องการดู lineup performer นั้น Performer สามารถกดปุ่มดูข้อมูลที่ต้องการและระบบจะแสดงชื่อ Performer ที่เกี่ยวข้อง 
8. หากต้องการดูรายละเอียดเพลงอื่น ๆ 
    - Performer กดย้อนกลับ 
    - ระบบนำ Performer กลับไปยังหน้ารายการเพลง 
9. Performer ทำซ้ำขั้นตอน 5 - 7 
10. Use case สิ้นสุดเมื่อ userไม่ต้องการดูเพลงอื่นๆแล้ว 

#### 10. Exceptions
**เพลงยังไม่ได้ถูกเพิ่มลงไปในระบบ**
1. User เลือก "Event" จากหน้า Main Menu 
    - ระบบแสดงรายการงานแสดงดนตรีทั้งหมด 
2. User เลือกงานแสดงดนตรีจากรายการที่มี 
    - ระบบทำการแสดงหน้ารายละเอียดของ 
งานแสดงดนตรีที่เลือก 
3. User กดปุ่ม "ดูรายชื่อเพลง" 
    - ระบบแสดงรายชื่อเพลงที่ใช้ในการแสดงของงานแสดงดนตรีนั้น 
4. User เลือกเพลงที่ต้องการดูรายละเอียดเพลง 
5. User ไม่พบเพลงในระบบ 
    - User ทำการแจ้งปัญหาไปที่ Backstage 
ผ่านช่องทางส่วนตัว 
6. Backstage ดำเนินการตาม Use case: เพิ่มเพลงใน Queue เพื่อเพิ่มเพลงที่ต้องการ 
    - ระบบบันทึกเพลงใหม่และแสดงในรายการเพลงของงานแสดงดนตรีนั้น ๆ 
7. User เข้ามาดูรายละเอียดเพลงอีกครั้ง 
8. Use Case ดำเนินการตามปกติ
